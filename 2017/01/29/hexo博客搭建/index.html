Hexo 博客搭建

技术
  • hexo
  • 教程

　　本文旨在记录搭建Hexo博客的过程，希望对到访本页的友人一点帮助。

　　博主的博客由于一些原因前后重新搭建了好几次，也遇到过不少障碍，有时候这次记得的步骤下次就忘记了，特写一篇较为详尽的博客来记录，以便回访。

一、前期准备

  • 注册Github账号
  • 安装Git for Windows和Node.js （在官网安装即可，其中git可能需要翻墙）

Git for windows部署

首先将之前注册的github账号和邮箱初始化给电脑
桌面右键运行git bash，依次输入如下指令

git config –global user.name “yourname”
git config –global user.email “youremail”

二、SSH key的相关设置

Windows下配置SSH并连接到github、github配置SSH Key

  • 检查本机是否有SSH Key设置
    输入命令 cd ~/.ssh
  1. 若没有，则提示： No such file or director 继续后面步骤
  2. 若有，则进入 ~/.ssh路径，rm * 删除所有文件

使用Git Bash生成新的ssh key

  1. cd ~ 保证当前路径在”~”下
  2. ssh-keygen -t rsa -C "xxxx@yy.com" 填写真实的邮箱地址，建议选择注册github时的邮箱

添加ssh key到github

  1. 登录github，点击”▽” —— settings —— ssh keys —— add ssh key
  2. 复制id_rsa.pub(用TXT打开，全选)的公钥，添加到相应位置
  3. 测试ssh keys是否配置成功,输入指令

    ssh -T git@github.com

　　若出现 “Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access”，则设置成功

三、远程连接

将本地项目通过SSH push到github

  1. 创建目录

    mkdir test
    cd test

  2. 初始化

    git init

  3. 创建 hello.md文件

    echo “这是一次测试test ssh key” > hello.md

  4. 提交到本地

    git add .
    git commit -m “add hello.md” #提交记录说明

  5. 提交到github

    git remote add origin ‘粘贴复制test ssh key的ssh路径’ #
    git push -u origin master

Hexo部署

　　首先在GitHub上创建一个项目，项目名为username.github.io，其中username为GitHub的账号名称。

  • 执行命令

    npm install hexo -g #全局安装hexo

　　第一次执行这个操作时，卡在安装过程。后卸载现有node.js并在官网下载最新版本，安装后依然无用。后来在寝室完成安装hexo。推测和下载网速有关。

  • 在合适的位置新建博客目录并转移到目录下

    mkdir hexo-blog
    cd hexo-blog

  • 初始化hexo

    hexo init

  • 启动

    hexo server

在浏览器中访问localhost:4000，若成则说明hexo配置成功(若按下ctrl+c则停止访问)

  • 将本地hexo与远程github关联

  • 修改项目根目录下的配置文件 _config.yml，用文本编辑器打开，修改最下面 deployer 属性为

    注意 type 、repository、branch 后的 : 为英文输入，且冒号后面有空格。

  • 安装hexo-deployer-git
    npm install hexo-deployer-git -S

  • 执行hexo g生成静态网页

  • 执行hexo d部署到远程github

　　至此可以直接通过https://username.github.io访问博客，目的达到！

　　可以申请自己喜欢的域名，和这个网址绑定，然后通过新的域名访问。
